# FamilyAiButler 多端 UI 设计

## 文档边界

本文档基于以下输入设计当前前端的一期多端 UI：

- `docs/api/openapi-summary.md`
- `docs/product/business-analysis.md`
- 当前前端实现：React Native + TypeScript 共享应用核心，Web 使用 Ant Design，桌面端通过 Tauri 包装同一套 React 页面。

设计只覆盖 OpenAPI 当前支撑的页面。没有接口支撑的数据不得使用假数据、mock 数据或静态伪造结果，只能展示空态、禁用态、隐藏入口，或标注为后端缺口。

所有业务请求必须通过 Nginx `/api` 前缀访问：

| 服务               | 前端路径规则                     |
|------------------|----------------------------|
| `family-core`    | `/api/base` + OpenAPI path |
| `family-uaa`     | `/api/uaa` + OpenAPI path  |
| `family-ai-qwen` | `/api/ai` + OpenAPI path   |

前端不得直连 `127.0.0.1:9527`、`/base`、`/uaa`、`/ai` 等 gateway 或服务内部地址。AI 图片理解当前按文档应调用
`/api/ai/ai/v1/image2Message`，但该路径需要联调确认。

## 现有 UI 语言

当前前端已经形成后台工具型视觉方向，应在一期扩展中保持一致：

- 桌面 Web/Tauri 使用固定左侧导航、顶部模块标题、内容卡片、表格和弹窗。
- 主要底色为浅灰蓝 `#f4f7fb` / `#f3f6fb`，主色为 `#2f6df6`，正文色为 `#1d2939`。
- 卡片、表格和筛选区以 8px 到 14px 圆角为主。新增交互控件优先使用 8px 圆角，避免进一步放大圆角。
- 页面以数据管理为主，应保持高信息密度和清晰层级，不做营销型首页、夸张 hero 或装饰性背景。
- 当前代码仍存在“总览、家庭成员、安全中心、设置”等入口，但 OpenAPI 未完整支撑这些页面。后续实现时应将越界入口改为明确空态或隐藏。

## 信息架构

### 一期导航

桌面 Web/Tauri 左侧导航建议顺序：

| 导航项     | 一期状态              | 说明                                   |
|---------|-------------------|--------------------------------------|
| 密码管理    | 可做                | 列表、详情、新增、编辑、删除、随机密码、强度检查             |
| 分类管理    | 可做                | 分类和分类类型 CRUD                         |
| AI 图片理解 | 可做但需联调            | 仅单次图片理解，不做聊天和历史                      |
| 个人资料    | 可做但依赖 `accountId` | 无 `me` 接口时展示空态和后端缺口                  |
| 权限与集成   | 受限支持              | RBAC/OAuth/JWK/授权决策按“调试型表单 + 缺口空态”设计 |
| 会话与设备   | 后端缺口              | 只有删除接口，无列表接口，默认空态                    |

认证页不放入登录后侧边栏。未登录时只允许访问登录、注册、找回密码、重置密码。

移动端底部主导航建议只保留 4 个高频入口：

| 移动端入口 | 对应页面               |
|-------|--------------------|
| 密码    | 密码管理               |
| 分类    | 分类管理               |
| AI    | AI 图片理解            |
| 我的    | 个人资料、权限与集成、会话与设备入口 |

权限与集成、会话与设备在移动端进入“我的”二级列表，不占用底部主导航。

### 页面结构原则

- 页面标题说明当前模块，不解释产品愿景。
- 筛选、主操作和批量操作放在列表上方，主表格或列表紧随其后。
- 详情、编辑、新增优先使用抽屉或弹窗，避免跳出当前上下文。
- 移动端不使用宽表格，降级为列表卡片或分组表单。
- 后端缺口必须用明确的空态文案说明“后端暂未提供接口”，不展示伪造数据。

## 多端布局规则

### 桌面 Web

| 区域     | 设计规格                                                                                       |
|--------|--------------------------------------------------------------------------------------------|
| 导航     | 左侧固定 `224px` Sider，深色背景，当前项使用主色填充。导航按钮必须是原生 `button` 或 Ant Design `Menu`，支持键盘聚焦。           |
| Header | 高度 `72px` 到 `76px`，左侧为页面标题和副标题，右侧为刷新、用户菜单或全局状态。不要放过多表单控件。                                  |
| 主体     | 最大宽度沿用当前 `1500px`，页面 padding `18px` 到 `24px`，模块间距 `16px`。                                  |
| Footer | 仅展示版本、环境和 API 前缀提示。不要把 gateway 内部地址展示为可编辑业务入口。                                             |
| 表单     | 筛选表单横向排列，字段最小宽度 `210px`。新增/编辑表单采用两列栅格，长文本独占一行。                                             |
| 表格     | 密码和分类列表使用 Ant Design `Table`，设置 `rowKey`、`loading`、`locale.emptyText`、横向滚动和分页。             |
| 抽屉/弹窗  | 新增/编辑建议用 `Modal` 或 `Drawer`。列表上下文较强的编辑用 `Drawer` 更合适，简单确认用 `Modal.confirm` 或 `Popconfirm`。 |

### Tauri 桌面端

Tauri 与 Web 复用同一套 React 页面，布局保持桌面 Web 规格，但需要额外处理窗口约束：

- 最小窗口宽度建议 `1024px`，低于 `960px` 时启用移动式横向导航或折叠导航。
- Header 不应依赖浏览器地址栏信息，环境标识放在 Footer 或用户菜单。
- 弹窗宽度使用 `min(720px, calc(100vw - 32px))`，避免小窗口溢出。
- 所有外链或站点字段只展示文本和复制动作，一期不要默认调用系统浏览器打开，除非产品确认。

### 移动端 React Native

移动端应使用同一业务信息架构，但组件降级为触控优先：

| 桌面组件           | 移动端降级                           |
|----------------|---------------------------------|
| `Layout.Sider` | 底部 Tab + “我的”二级列表               |
| `Header`       | 顶部安全区内标题栏，左侧返回或标题，右侧一个主操作       |
| `Table`        | `FlatList` / `SectionList` 卡片列表 |
| `Modal`        | 全屏表单页或底部 Sheet                  |
| `Drawer`       | 全屏详情页或底部 Sheet                  |
| `Input.Search` | 顶部搜索框，筛选项收进底部 Sheet             |
| `Popconfirm`   | 原生确认弹窗或底部确认 Sheet               |
| `Tabs`         | 分段控件或横向滚动 Tab                   |

移动端触控目标最小高度 `44px`。列表项主行展示名称、账号或类型，副行展示网址、分类、更新时间。危险操作不放在列表首屏，进入详情后展示。

## 屏幕设计建议

### 登录/认证

#### 支撑接口

| 功能      | 前端路径                                   |
|---------|----------------------------------------|
| 密码登录    | `POST /api/uaa/auth/login/password`    |
| 邮箱验证码登录 | `POST /api/uaa/auth/login/email-code`  |
| 短信验证码登录 | `POST /api/uaa/auth/login/sms-code`    |
| 注册      | `POST /api/uaa/account/register`       |
| 密码恢复    | `POST /api/uaa/auth/password/recovery` |
| 密码重置    | `POST /api/uaa/auth/password/reset`    |
| 当前会话退出  | `POST /api/uaa/auth/logout/current`    |
| 全部会话退出  | `POST /api/uaa/auth/logout/all`        |

#### 桌面 Web/Tauri 布局

- 使用居中认证面板，宽度 `380px` 到 `420px`。
- 左上或面板顶部展示 `FamilyAiButler` 品牌和图标，不需要大面积插画。
- 主 Tab 为“密码登录”和“验证码登录”。验证码登录可展示，但“获取验证码”按钮必须禁用，并提示“后端暂未提供验证码发送接口”。
- 登录表单字段顺序：账号、密码、设备名称、设备指纹。设备字段默认折叠在“高级选项”中，避免干扰主流程。
- 注册页字段顺序：用户名、邮箱、手机号、密码、确认密码。邮箱和手机号可选时不要做必填。
- 找回密码页拆成两个步骤：发起恢复、重置密码。恢复成功后根据后端返回值提示下一步，不假设验证码已发送。

#### 移动端布局

- 使用全屏认证页，顶部留安全区，表单从屏幕上方约 `20%` 开始。
- 登录方式使用分段控件，不使用多列布局。
- 主按钮固定在表单下方，不固定在屏幕底部，避免键盘遮挡。

#### 交互和状态

- 登录提交后按钮进入 loading，禁止重复提交。成功后立即拉起登录后默认页“密码管理”。
- `401` 或 token 失效时清理本地 token，跳转登录页，并通过全局提示展示“登录状态已失效，请重新登录”。
- 登录失败保留账号字段，清空密码字段。
- 验证码发送能力缺失时，验证码登录入口可以保留为 disabled 状态；不要隐藏到用户误以为产品不支持验证码。
- 表单校验错误在字段下方展示，提交失败使用 `Alert` 或 toast，不只依赖控制台。

#### 可访问性

- 焦点顺序：账号 -> 密码 -> 高级选项 -> 登录 -> 注册/忘记密码链接。
- Enter 在密码框触发登录。
- 登录方式 Tab 必须可左右方向键切换。
- 错误提示容器使用 `role="alert"`，移动端用屏幕阅读器可读的状态提示。

### 密码管理

#### 支撑接口

| 功能     | 前端路径                                                        |
|--------|-------------------------------------------------------------|
| 列表     | `GET /api/base/password/password/list`                      |
| 分页列表   | `GET /api/base/password/password/list/{pageNum}/{pageSize}` |
| 详情     | `GET /api/base/password/{id}`                               |
| 业务主键详情 | `GET /api/base/password/business/{businessId}`              |
| 新增     | `POST /api/base/password/password/add`                      |
| 修改     | `PUT /api/base/password`                                    |
| 删除     | `DELETE /api/base/password`                                 |
| 随机密码   | `GET /api/base/password/generate/{passwordLength}`          |
| 强度检查   | `GET /api/base/password/checkStrength/{password}`           |
| 合法性检查  | `GET /api/base/password/checkValid/{password}`              |

#### 桌面 Web/Tauri 布局

- 页面顶部保留筛选卡片：关键词、分类、查询、重置、新增账号。
- 指标卡片只展示接口真实可计算的数据：账号总数、当前页数量、分类数量、当前页高风险数量。不要展示全局安全评分、泄露检测等无接口数据。
- 列表使用 `Table`，列顺序建议：名称、网址、账号、分类、安全等级、收藏、最近更新时间、操作。
- 密码字段默认不在表格中明文展示。详情/编辑抽屉内用 `Input.Password`，提供“复制”动作时必须二次确认或短时明文展示。
- 新增/编辑使用右侧 `Drawer`，桌面宽度 `640px` 到 `720px`。当前实现如继续用 `Modal`，宽度保持 `720px`，内容两列。
- 删除单条用 `Popconfirm`，批量删除在有勾选时启用。

#### 移动端布局

- 顶部为搜索框，筛选按钮打开底部 Sheet。
- 密码列表用卡片：名称 + 分类标签为主行，账号和网址为副行，右侧只放“更多”图标。
- 新增按钮使用右下角固定圆形按钮或 Header 右侧按钮，两者择一，不重复。
- 详情和编辑使用全屏页面。删除按钮放在页面底部危险区。

#### 交互和状态

- 列表加载使用 `Table` loading 或移动端骨架屏，超过 `300ms` 显示。
- 查询按钮点击后保留当前筛选值，分页重置到第一页。
- 保存成功后关闭抽屉并刷新当前列表；新增成功回到第一页。
- 删除成功后清空选中项；若当前页被删空且页码大于 1，应回退上一页重新查询。
- 随机密码生成成功后填入密码字段，并立即调用强度检查。
- 强度检查采用防抖 `400ms`，输入为空时不请求接口。

#### 空态和错误态

- 无数据：`Empty` 文案“后端暂无账号数据”，主操作为“新增账号”。
- 接口错误：列表区域顶部展示 `Alert`，保留刷新按钮。不要用假数据兜底。
- 请求契约不清晰：新增接口 query/body 并存、GET 列表 body 风险，需要在实现中集中标注为联调风险。

#### 可访问性

- 表格行操作顺序：查看 -> 编辑 -> 更多 -> 删除确认。
- 勾选框必须有可读标签，如“选择账号 {name}”。
- 复制密码成功后用 `aria-live` 或 toast 提示。
- 密码显示/隐藏按钮必须有明确标签，不能只显示图标。

### 分类管理

#### 支撑接口

| 功能     | 前端路径                                           |
|--------|------------------------------------------------|
| 分类类型列表 | `GET /api/base/category/type/list`             |
| 分类列表   | `GET /api/base/category/list`                  |
| 新增分类   | `POST /api/base/category/category`             |
| 更新分类   | `PUT /api/base/category/category`              |
| 分类详情   | `GET /api/base/category/category/{id}`         |
| 删除分类   | `DELETE /api/base/category/category/{id}`      |
| 新增分类类型 | `POST /api/base/category/category/type`        |
| 更新分类类型 | `PUT /api/base/category/category/type`         |
| 分类类型详情 | `GET /api/base/category/category/type/{id}`    |
| 删除分类类型 | `DELETE /api/base/category/category/type/{id}` |

#### 桌面 Web/Tauri 布局

- 分类管理页使用两个 Tab：分类、分类类型。
- 分类 Tab 包含搜索、父级 ID 过滤、新增分类、分类表格。
- 分类类型 Tab 包含搜索、新增类型、类型表格。
- 分类表格列：分类名称、父级 ID、说明、更新时间、操作。
- 分类类型表格列：类型名称、说明、创建时间、更新时间、操作。
- 不设计拖拽排序、树形移动、合并分类，因为后端未提供接口。

#### 移动端布局

- 顶部使用分段控件切换“分类 / 类型”。
- 列表卡片展示名称、说明、父级 ID 或更新时间。
- 新增/编辑使用全屏表单，删除放在详情页危险区。

#### 交互和状态

- 新增分类时父级 ID 使用数字输入框，默认空或 `0`，不要展示前端伪造树选择。
- 如果后端返回分类列表为空，父级选择只能禁用或允许手动输入 ID。
- 删除前必须二次确认。删除成功后刷新当前 Tab。
- 分类类型删除可能影响分类展示，成功后同时刷新分类和分类类型列表。

#### 可访问性

- Tab 顺序：分类 Tab -> 类型 Tab -> 搜索 -> 新增 -> 表格。
- 删除确认弹窗焦点默认在“取消”，避免误删。
- 表单字段标签必须与接口字段一致：名称、说明、父级 ID、类型名称。

### AI 图片理解

#### 支撑接口

| 功能   | 前端路径 |
|------|------|
| 图片理解 | `GET |POST|PUT|DELETE|PATCH /api/ai/ai/v1/image2Message` |

一期 UI 只使用一个主调用方式，建议优先使用 `POST /api/ai/ai/v1/image2Message`。参数为 query `file` string。当前 OpenAPI 未说明
`file` 是 URL、base64、文件 ID 还是 multipart 文件，因此页面必须把输入设计为“图片参数字符串”，不要直接承诺文件上传。

#### 桌面 Web/Tauri 布局

- 页面使用左右两栏：
    - 左侧：输入卡片，包含 `file` 字符串输入、调用方式只读说明、提交按钮。
    - 右侧：结果卡片，展示后端返回文本。
- 如果产品希望放上传组件，一期只能禁用 `Upload`，文案为“后端暂未提供标准上传契约”。
- 不展示聊天气泡、历史记录、模型选择器、图片生成入口。

#### 移动端布局

- 单列结构：输入卡片、提交按钮、结果卡片。
- 结果文本支持长按复制。

#### 交互和状态

- 输入为空时提交按钮禁用。
- 请求中显示结果区骨架屏，按钮 loading。
- 成功后结果区展示纯文本，提供复制按钮。
- `401` 进入全局未认证处理。
- `404` 或路径错误展示“AI 接口路径需联调确认”，并保留请求路径信息。
- 参数错误展示“图片参数格式不被后端接受”，不要自动切换成上传假流程。

#### 可访问性

- 焦点顺序：图片参数输入 -> 提交 -> 复制结果。
- 结果区使用可选择文本；复制按钮要有文本标签或 `aria-label`。
- loading 状态应对屏幕阅读器宣告“正在识别图片”。

### 个人资料

#### 支撑接口

| 功能           | 前端路径                                       |
|--------------|--------------------------------------------|
| 创建 Profile   | `POST /api/uaa/profile`                    |
| 更新 Profile   | `PUT /api/uaa/profile`                     |
| 查询账号 Profile | `GET /api/uaa/profile/account/{accountId}` |
| 删除 Profile   | `DELETE /api/uaa/profile/{profileId}`      |

#### 设计建议

- 页面标题为“个人资料”，副标题为“维护账号附属资料”。
- 如果当前 token 或登录响应能稳定提供 `accountId`，页面加载时查询 Profile 列表。
- 如果拿不到 `accountId`，主区域展示后端缺口空态：`当前 OpenAPI 未提供 me 接口，无法稳定获取当前账号 ID`。
- 表单字段只使用 `profileId`、`accountId`、`nickname`、`avatar`、`language`、`region`、`profileType`。
- `avatar` 只做 URL 或字符串输入，不做上传。
- 不设计改密码、绑定邮箱、绑定手机号，因为没有接口。

#### 多端布局

- 桌面端用左侧 Profile 列表 + 右侧编辑表单。无数据时右侧展示新建表单。
- 移动端用 Profile 列表进入详情编辑页。

#### 状态和可访问性

- 账号 ID 缺失时所有保存按钮禁用。
- 保存成功后刷新 Profile 列表。
- 删除 Profile 前二次确认，确认焦点默认在取消。

### 权限与集成

#### 支撑接口

| 功能              | 前端路径                                                 |
|-----------------|------------------------------------------------------|
| 授权决策            | `POST /api/uaa/authorization/decide`                 |
| JWK             | `GET /api/uaa/.well-known/jwks.json`                 |
| 新增或更新角色         | `POST /api/uaa/rbac/roles`                           |
| 新增或更新资源         | `POST /api/uaa/rbac/resources`                       |
| 绑定角色资源          | `POST /api/uaa/rbac/role-resources`                  |
| 绑定账号角色          | `POST /api/uaa/rbac/account-roles`                   |
| 查询账号权限          | `GET /api/uaa/rbac/accounts/{accountId}/permissions` |
| OAuth Client 列表 | `GET /api/uaa/oauth-clients`                         |
| 新增 OAuth Client | `POST /api/uaa/oauth-clients`                        |
| OAuth Client 详情 | `GET /api/uaa/oauth-clients/{clientId}`              |

#### 设计建议

权限能力当前偏底层原子接口。不要设计完整权限后台，不要展示角色列表、资源列表、账号列表或绑定关系列表，除非后端补接口。

桌面端页面使用四个 Tab：

| Tab          | 一期表现                                                          |
|--------------|---------------------------------------------------------------|
| 授权决策         | 可做表单：accessToken、resourceService、resourcePath、action，右侧展示返回结果 |
| 账号权限查询       | 可做表单：accountId、resourceType，展示真实返回列表或空态                       |
| RBAC 写入      | 受限表单：角色、资源、绑定操作分段展示；无列表回显                                     |
| OAuth Client | 可做列表、新增、详情；更新和删除展示后端缺口                                        |

移动端将四个 Tab 降级为“我的”内的分组列表，每个能力进入独立页面，避免复杂横向 Tab。

#### 状态和可访问性

- RBAC 写入表单提交后只展示后端返回结果，不伪造列表刷新。
- OAuth Client 列表为空时展示“后端暂无 OAuth Client 数据”。
- 缺少删除/更新接口的按钮不展示；如果产品要求露出，则必须 disabled 并标注“后端未提供接口”。
- 权限调试结果区域使用只读代码块，支持复制。

### 会话与设备

#### 支撑接口

| 功能   | 前端路径                                  |
|------|---------------------------------------|
| 删除会话 | `DELETE /api/uaa/session/{sessionId}` |
| 删除设备 | `DELETE /api/uaa/device/{deviceId}`   |

当前 OpenAPI 没有会话列表和设备列表接口，因此一期不做可管理列表。

#### 设计建议

- 页面默认展示后端缺口空态：`当前后端仅提供删除接口，未提供会话/设备列表，无法展示可管理数据`。
- 可选提供“手动删除”调试表单：输入 sessionId 或 deviceId 后调用删除接口。
- 手动删除表单默认折叠，避免普通用户误用。
- 移动端只展示空态，不建议提供调试表单。

## 组件选型

### Ant Design Web/Tauri

| 场景    | 组件                                                         |
|-------|------------------------------------------------------------|
| 应用壳   | `Layout`, `Menu` 或当前自定义 `button` 导航                        |
| 页面标题  | `Typography.Title`, `Typography.Text`                      |
| 数据卡片  | `Card`, `Statistic` 或当前 `MetricCard`                       |
| 筛选    | `Form`, `Input.Search`, `Select`, `Button`, `Space`        |
| 表格    | `Table`, `Tag`, `Tooltip`, `Dropdown`                      |
| 新增/编辑 | `Drawer` 优先，简单场景可继续使用 `Modal`                              |
| 删除确认  | `Popconfirm`，批量危险操作用 `Modal.confirm`                       |
| 状态反馈  | `Spin`, `Skeleton`, `Alert`, `Result`, `Empty`, `message`  |
| 认证    | `Form`, `Input`, `Input.Password`, `Tabs`, `Alert`         |
| AI 结果 | `Input.TextArea`, `Card`, `Typography.Paragraph`, `Button` |

不要新增设计系统组件，除非出现以下情况：

- 跨三个以上页面重复出现同一种状态展示时，可抽取 `BackendGapEmpty`。
- 所有受保护页面都需要统一 401 处理时，可抽取 `AuthGuard`。
- 列表页都需要相同加载/错误/空态组合时，可抽取 `DataState`.

除此之外，优先复用 Ant Design 和当前 `FieldBlock`、`MetricCard` 一类轻量组件，避免过早建立复杂组件库。

### 移动端 React Native

移动端不直接照搬 Ant Design 表格。建议使用：

- 页面壳：`SafeAreaView`、`View`、`ScrollView`。
- 列表：`FlatList`、`SectionList`。
- 表单：项目已有表单组件优先；没有时先用 `TextInput`、`Pressable`、受控状态实现。
- 弹层：底部 Sheet 或全屏页面。若项目引入跨端组件库，必须保证键盘避让和屏幕阅读器标签。
- 状态：骨架屏、空态文本、错误提示条、禁用按钮。

## 统一交互状态

| 状态      | 桌面 Web/Tauri                                   | 移动端                   |
|---------|------------------------------------------------|-----------------------|
| 初次加载    | 卡片内 `Skeleton` 或 `Table loading`，超过 `300ms` 显示 | 列表骨架屏或居中 loading      |
| 局部刷新    | 刷新按钮 loading，保留旧数据                             | 下拉刷新或按钮 loading，保留旧数据 |
| 空数据     | `Empty` + 可用主操作                                | 空态插图可省略，展示主操作按钮       |
| 接口错误    | `Alert` 展示错误摘要和刷新按钮                            | 顶部错误条 + 重试按钮          |
| 401 未认证 | 清 token，跳登录页，展示一次提示                            | 清 token，跳登录页，展示一次提示   |
| 403 无权限 | `Result status="403"` 或页面内禁用态                  | 空态页说明无权限              |
| 后端未提供数据 | `BackendGapEmpty`，说明缺接口名和影响                    | 空态说明，不展示调试入口          |
| 保存中     | 主按钮 loading，禁用取消以外的重复提交                        | 主按钮 loading，禁用重复提交    |
| 删除确认    | `Popconfirm`/`Modal.confirm`，焦点默认取消            | 原生确认或 Sheet，危险按钮靠后    |

错误提示不得只用 `message.error`。列表、详情、表单页都要在上下文中保留可见错误，便于用户重试和开发联调。

## 焦点顺序和键盘交互

### 全局

- Tab 顺序遵循视觉顺序：导航 -> Header 操作 -> 筛选区 -> 主操作 -> 数据区 -> 分页。
- 当前导航项必须有 `aria-current="page"` 或等价状态。
- 所有图标按钮必须有文字、tooltip 或 `aria-label`。
- 弹窗打开后焦点进入第一个可编辑字段，关闭后焦点回到触发按钮。
- ESC 关闭非破坏性弹窗；删除确认需要用户明确选择。

### 表格

- 表格行内操作使用按钮，不使用不可聚焦的文本。
- `Dropdown` 打开后上下方向键可选择菜单项，ESC 关闭。
- 分页控件保持 Ant Design 默认键盘能力。

### 移动端

- 触控目标不小于 `44px`。
- 文本输入必须设置明确 label 或 accessibilityLabel。
- 页面切换后将屏幕阅读器焦点移动到页面标题。
- 底部 Sheet 打开时禁止背景内容被屏幕阅读器继续访问。

## 数据和接口约束

- API client 统一接收相对路径或 `/api` 前缀地址，不允许业务代码拼接 `127.0.0.1:9527`。
- `Authorization` 注入由统一请求层处理，页面组件不直接操作请求头。
- `Result<T>` 的 `data` 类型在 OpenAPI 中不可靠，页面解析要防御 `data.records`、`data.list`、`records`
  等实际返回形态，但不能把解析失败转为假数据。
- 登录、注册、密码恢复、密码重置的鉴权规则与 OpenAPI 全局 security 冲突，实现时以运行时联调结果为准，并在 API 层集中配置白名单。
- AI 图片理解路径和 `file` 参数语义必须作为联调任务，不在 UI 中隐含承诺上传能力。

## 实现指导

### 推荐实施顺序

1. 建立统一认证状态：登录页、token 保存、401 拦截、退出登录。
2. 调整导航为 OpenAPI 支撑的页面集合，未支撑入口进入空态或隐藏。
3. 完善密码管理：列表、筛选、新增/编辑、删除、随机密码、强度检查。
4. 完善分类管理：分类和分类类型双 Tab。
5. 增加 AI 图片理解最小联调页。
6. 增加个人资料页面，并在缺 `accountId` 时展示后端缺口空态。
7. 增加权限与集成、会话与设备的受限页面或空态。

### 不建议立即做的事项

- 不做完整总览大盘，因为 OpenAPI 没有跨域统计接口。
- 不做家庭成员页面，因为没有家庭成员、家庭空间或邀请接口。
- 不做通用 AI 聊天，因为当前只有图片转文本描述。
- 不做会话/设备列表，因为只有删除接口。
- 不做完整 RBAC 后台，因为缺列表、分页、删除和账号列表。
- 不新增主题系统或复杂设计 token，当前颜色、间距和 Ant Design token 已够用。

### 可以新增的轻量组件

| 组件                    | 必要性  | 说明                                   |
|-----------------------|------|--------------------------------------|
| `AuthGuard`           | 需要   | 统一保护受认证页面和 401 跳转                    |
| `BackendGapEmpty`     | 需要   | 所有接口缺口页面统一展示                         |
| `DataState`           | 可选   | 统一 loading/error/empty 包裹，密码和分类列表可复用 |
| `PageHeader`          | 可选   | 如果多个页面重复标题、副标题、右侧操作                  |
| `ResponsiveListTable` | 暂不需要 | 桌面表格与移动列表差异较大，先分端实现更清晰               |

新增组件应保持浅层调用，不要把简单页面拆成多层私有组件链。

## 验收检查

- 页面只覆盖 OpenAPI 当前支撑能力。
- 所有请求路径以 `/api` 开头。
- 没有业务请求直连 `127.0.0.1:9527`。
- 密码、分类、AI、UAA 页面都有 loading、empty、error、401 和后端缺口处理。
- 桌面 Web/Tauri 与移动端分别定义导航、Header、主体、Footer、表单、表格/列表、抽屉/弹窗差异。
- Ant Design 组件选型清晰，移动端降级策略清晰。
- 焦点顺序、键盘操作和屏幕阅读器提示有明确要求。
- 未被接口支撑的能力不展示假数据，不承诺完整功能。
